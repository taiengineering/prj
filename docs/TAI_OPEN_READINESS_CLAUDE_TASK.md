## 목적

TAI를 실제 오픈 가능한 산업안전 SaaS 상태로 만들기 위한 작업지시서입니다.

## 오픈 가능 상태 정의

TAI의 오픈은 단순 기능 완성이 아니라 고객이 실제로 가입하고, 사용하고, 결제할 수 있는 상태를 의미합니다.

오픈 가능 조건:

1. 고객이 혼자 가입 가능
2. 고객이 사용법을 이해 가능
3. 실제 현장 업무에 즉시 사용 가능
4. 오류 없이 데이터 저장 가능
5. 유료 결제를 받을 수 있음
6. 결제 후 플랜/권한이 정상 적용됨
7. 고객지원 및 운영자가 장애 대응 가능

## 소비자 관점 핵심 검증

고객은 기술 구조를 모릅니다. 따라서 첫 사용 경험에서 아래가 명확해야 합니다.

### 첫 화면

첫 진입 시 다음 행동이 바로 보여야 합니다.

- 현장 생성
- TBM 시작
- 점검 시작
- 작업자 등록

복잡한 메뉴, 내부 용어, 개발자용 표현은 제거합니다.

### 모바일 UX

산업안전 현장은 모바일 중심입니다.

검증 기준:

- 버튼이 충분히 큼
- 장갑 낀 상태에서도 누르기 쉬움
- 입력 단계가 짧음
- 스크롤이 과도하지 않음
- 느린 네트워크에서도 사용 가능
- PDF 생성/확인이 모바일에서 가능

### 고객 가치 표현

랜딩페이지와 대시보드에서 다음 가치가 즉시 보여야 합니다.

- TBM 문서 작성시간 감소
- 점검 누락 방지
- 법정 문서 자동 보관
- 중대재해 대응 기록 확보
- 관리자 업무시간 절감

- PDF 출력 시 타임스탬프
- 법령/점검 기준 버전 기록

## 오픈 전 차단 이슈 기준

아래 문제가 하나라도 있으면 유료 오픈 금지입니다.

- 가입 불가
- 로그인 불안정
- 데이터 저장 실패
- 타 회사/타 현장 데이터 노출
- PDF 생성 실패
- 결제 성공 후 플랜 미적용
- 결제 실패 후 오류 화면 방치
- 모바일에서 핵심 기능 사용 불가
- 개인정보처리방침/이용약관 없음
- 관리자 대응 화면 없음

## 최종 원칙

TAI는 단순 체크리스트 앱이 아니라 산업안전 운영 SaaS입니다.

따라서 모든 작업은 아래 기준으로 판단합니다.

1. 고객이 이해 가능한가?
2. 현장에서 빠르게 사용 가능한가?
3. 법적 문서로 신뢰 가능한가?
4. 결제할 가치가 명확한가?
5. 운영자가 장애와 고객 문의를 처리할 수 있는가?

이 기준을 통과하지 못하면 오픈하지 않습니다.
